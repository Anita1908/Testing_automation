Баг #1: Неправильная обработка null-значений в методе capitalize()

Название: Неверная работа с типом None в методе capitalize()
Среда: Python 3.13.7, PyTest 8.4.2
Шаг воспроизведения:Вызвать метод capitalize() с аргументом None.
Ожидаемый результат: Возвращение оригинального значения (None) или подходящей замены (например, пустой строки).
Фактический результат: Возникает исключение AttributeError: 'NoneType' object has no attribute 'capitalize'.
Приоритет: Medium
Серьезность: Minor
Комментарии: Требуется ввести дополнительную проверку на None и возврат подходящего значения.


Баг #2: Проблемы с None в методе trim()

Название: Метод trim() падает при получении None
Среда: Python 3.13.7, PyTest 8.4.2
Шаг воспроизведения:Вызвать метод trim() с аргументом None.
Ожидаемый результат: Возвращение оригинальной строки или подходящей замены (например, пустой строки).
Фактический результат: Происходит исключение AttributeError: 'NoneType' object has no attribute 'startswith'.
Приоритет: Medium
Серьезность: Major
Комментарии: Нужно добавить проверку на корректность ввода и возвратить подходящее значение в случае неправильного формата.



Баг #3: Неверная логика поиска символов в методе contains()

Название: Метод contains() ошибочно находит слово целиком
Среда: Python 3.13.7, PyTest 8.4.2
Шаг воспроизведения:Вызвать метод contains() с параметрами "SkyPro" и "Pro".
Ожидаемый результат: Метод должен вернуть False, так как ищет точное вхождение символа, а не части строки.
Фактический результат: Метод возвращает True, несмотря на отсутствие точного совпадения.
Приоритет: High
Серьезность: Critical
Комментарии: Следует реализовать чёткое разделение поиска символов и целых слов, учитывая длину переданного символа.


Баг #4: Ошибка индексации в методе delete_symbol()

Название: Неверная обработка нулевых значений в методе delete_symbol()
Среда: Python 3.13.7, PyTest 8.4.2
Шаг воспроизведения:Вызвать метод delete_symbol() с аргументом None.
Ожидаемый результат: Не удалять символ и вернуть оригинальный текст.
Фактический результат: Возникает исключение TypeError: index() argument 1 must be str, not None.
Приоритет: Medium
Серьезность: Major
Комментарии: Требуется добавить проверку на корректность входящего аргумента и возвращение исходной строки в случае отсутствия правильного символа.
